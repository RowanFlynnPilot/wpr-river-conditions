# CLAUDE.md — WPR River & Lake Conditions Widget

## Overview
Real-time river and stream conditions dashboard for central Wisconsin, built for Wausau Pilot & Review. Displays USGS stream gauge data with NWS flood thresholds, NWS flood alerts, and WVIC reservoir levels in a branded, embeddable widget.

**Live URL**: `https://rowanflynnpilot.github.io/wpr-river-conditions/`
**WordPress embed**: `<iframe src="https://rowanflynnpilot.github.io/wpr-river-conditions/" width="100%" height="900" frameborder="0"></iframe>`

## Architecture
Same pattern as other WPR widgets (gas prices, weather radar, adopt widget):
- **Python scraper** → **GitHub Actions** (cron every 30 min) → **GitHub Pages** → **WordPress iframe**
- Frontend: React + Vite, deployed as static files
- Data: JSON file generated by `scripts/fetch_data.py`, served from `/data/river-data.json`

## Data Sources

### USGS Water Services API (primary — gauge readings)
- **IV Endpoint**: `https://waterservices.usgs.gov/nwis/iv/` (instantaneous values, ~15 min intervals)
- **DV Endpoint**: `https://waterservices.usgs.gov/nwis/dv/` (daily values for 7-day sparklines)
- **Format**: JSON
- **Auth**: None (public API, requires User-Agent header)
- **Rate limits**: Reasonable use; no hard limit documented
- **⚠️ Migration note**: Legacy WaterServices API decommissions early 2027. New API at `https://api.waterdata.usgs.gov/` uses OGC API Features standard. Plan migration for late 2026.

### NWS NWPS API (flood thresholds and category cross-check)
- **Endpoint**: `https://api.water.noaa.gov/nwps/v1/gauges/{nws_lid}`
- **Format**: JSON
- **Auth**: None
- **Provides**: Official NWS flood stage thresholds (action/minor/moderate/major), current flood category, historic crests
- Replaced the old AHPS system in March 2024

### NWS Alerts API (active flood warnings)
- **Endpoint**: `https://api.weather.gov/alerts/active?zone=WIC073`
- **Format**: GeoJSON
- **Auth**: None (requires User-Agent)
- Filtered to flood-related events only (flood, flash flood, river, hydrologic)

### WVIC Reservoir Data (Phase 2 — not yet implemented)
- **Source pages**: `https://wvic.com/Content/{Name}_Flows.cfm`
  - `Rainbow_Flows.cfm`, `Willow_Flows.cfm`, `Rice_Flows.cfm`, `Spirit_Flows.cfm`, `Eau_Pleine_Flows.cfm`
- **Reservoirs**: Rainbow, Willow, Spirit, Eau Pleine, Rice, Lake Wausau
- **Approach**: Needs Playwright scraping — data renders via JavaScript/embedded chart images, not scrapeable HTML
- The main data hub page is `https://wvic.com/Content/Data--Reports.cfm`

**Important**: The Rothschild gauge (primary Wausau gauge) uses gage height in feet. The *separate* NWS gauge WUUW3 ("Wisconsin River below Wausau Dam") reports in *elevation* above sea level (flood stage = 1167 ft NAVD88). These are different measurement systems for overlapping but different locations. Our widget uses ROTW3 (Rothschild, gage height) which aligns with the USGS data.

## Flood Status Logic
Four-tier system matching NWS standard categories:
- **Normal**: Below action stage
- **Action**: At or above action stage (monitoring threshold)
- **Minor Flood**: At or above minor flood stage
- **Moderate**: At or above moderate flood stage
- **Major Flood**: At or above major flood stage

The scraper also fetches the NWS's own flood category from the NWPS API as a cross-check (`nws_flood_category` field).

## GitHub Setup — Special Files Checklist

These files are critical and easy to lose during upload (especially hidden dotfiles on Mac/Windows):

| File | Purpose | Easy to miss? |
|------|---------|---------------|
| `.github/workflows/deploy.yml` | Automated data fetch + deploy | **YES** — hidden folder, won't show in Finder/Explorer by default |
| `.gitignore` | Keeps node_modules and generated data out of repo | YES — hidden dotfile |
| `public/.nojekyll` | Tells GitHub Pages to skip Jekyll processing | YES — hidden dotfile |
| `vite.config.js` | Sets `base: '/wpr-river-conditions/'` for GH Pages paths | No |

**To reveal hidden files for upload:**
- **Mac**: In Finder, press `Cmd+Shift+.`
- **Windows**: In Explorer, View → Show → Hidden items

**GitHub Pages settings**: Settings → Pages → Source: "GitHub Actions"
